## Content Insights
While using Docker, I was slowly starting to understand the workflow of using local environments and then pushing to a live server. I learnt that it works by using containers that 'package' my WordPress app. The .yml docker file contains 'instructions' to build the image. The image contains the WordPress code that helps my application function. Docker will be very useful for locally hosting other cms websites as well. I have found that it is important to use Docker so that it eases the way environments work. On the other hand, vagrant seems like a worthy option, which involves virtual machines. However, I found Docker to be more simple, since this is the first time I'm doing this sort of thing.

I also found it important to consider the site owner's use of the website, as well as the user's use. There could be a 'manager' page as well that allows the owner of the company/establishment to manage their assets or view their company statistics or deal with customers.
